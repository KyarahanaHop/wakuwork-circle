## 衝突解決ルール

**本ファイル（decisions.md）と他SSoTファイルの間で衝突・矛盾が生じた場合:**

```
優先順位: core.md > decisions.md > 他SSoTファイル
```

- `core.md` が仕様の最終権威
- 本ファイルは「決定の経緯と理由」を記録する補助ドキュメント
- 矛盾を発見したら `core.md` を正として他を修正する

## 決定一覧

### D-001: 応援金額表示ポリシー

**決定日**: 2026-02-02  
**ステータス**: 確定

| 閲覧者 | 金額表示 | 備考 |
|--------|----------|------|
| **視聴者** | **あり**（個別イベント） | 時系列固定、ランキングなし、合計なしで煽り防止 |
| **配信者** | **あり**（全額） | 収益把握・会計のため |

**絶対禁止事項（煽り防止）**:
- 合計/累計表示
- ランキング（1位/2位/3位）
- Top支援者表示
- 金額による並べ替え（ソート）
- 「もっと応援しろ圧」を誘発する演出・UI

**許可**:
- 個別イベントでの金額表示
- 時系列順での一覧表示（新しい順のみ）

**参照**: `core.md` Section 7.4

### D-002: 合言葉（Passphrase）設定

**決定日**: 2026-02-02  
**ステータス**: 確定

| 設定項目 | 仕様 |
|----------|------|
| ON/OFF | 配信者が切り替え可能 |
| 初期値 | **ON**（合言葉必須） |
| OFF時の挙動 | **初回参加者は自動でロビー経由**（承認待ち強制） |

**セーフティロック（UI警告）**:
- 合言葉OFF + 承認OFF を同時に設定する場合、UIに**強い警告**を表示
- 警告文例: 「誰でも即入室できる状態になります。荒らし対策として推奨しません。」
- 設定は許可するが、リスクを明確に伝える

**参照**: `flows.md` Section 1.2

### D-004: 閲覧専用ゲスト

**決定日**: 2026-02-02  
**ステータス**: 確定

| 項目 | 仕様 |
|------|------|
| 閲覧だけゲスト | **なし**（提供しない） |
| 入室要件 | **全員OAuth必須**（Discord） |
| 外部閲覧 | **提供しない**（ウォッチャー対策） |

**理由**:
- ウォッチャー（覗き見）による治安悪化を防止
- 全員がアカウント紐付きであることでモデレーションが機能する
- 匿名閲覧者がいると配信者・参加者が不安になる

**参照**: `core.md` Section 3

### D-005: リアルタイム通信方針（MVP）

**決定日**: 2026-02-02  
**ステータス**: 確定

| 機能 | MVP方式 | 将来検討 |
|------|---------|----------|
| タイマー同期 | `sessionStartedAt`共有 + **5〜10秒ポーリング補正** | WebSocket |
| Presence（在席） | ポーリング | WebSocket/SSE |
| スタンプ | ポーリング（擬似同席感） | WebSocket |
| 完了状態 | ポーリング | WebSocket |

**理由**:
- MVPはシンプルさを優先
- ポーリング間隔で「擬似同席感」は十分達成可能
- WebSocket等の複雑性はスケール要件が明確になってから導入

**参照**: `open-questions.md` から移動（決定済み）
