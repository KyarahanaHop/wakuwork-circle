## 1. 技術選定

### 1.1 リアルタイム通信方式

> **✅ 決定済み** → `decisions.md` D-005 参照  
> MVP: ポーリング（5〜10秒間隔）

### 1.2 バックエンド方式

| 選択肢 | 特徴 | 検討ポイント |
|--------|------|--------------|
| Next.js API Routes | シンプル、一体型 | スケールに限界 |
| BaaS (Supabase等) | リアルタイム標準装備 | ベンダーロック |
| 自前サーバー (Node/Go) | 柔軟、運用コスト大 | チーム規模に依存 |

**決定基準**: 開発リソース、スケール要件、運用体制

### 1.3 データベース

| 選択肢 | 特徴 | 検討ポイント |
|--------|------|--------------|
| PostgreSQL | 堅牢、SQL標準 | 自前運用 or マネージド |
| Supabase (PostgreSQL) | マネージド、リアルタイム | BaaS選択時 |
| PlanetScale (MySQL) | サーバーレス、スケール | MySQL互換が必要 |

**決定基準**: BaaS選択、コスト、スケール要件
