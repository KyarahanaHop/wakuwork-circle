## 2. データ保持ポリシー

### 2.1 参加履歴（Pro: 30日）

保持される情報:
- 参加日時
- 参加セッション
- 参加者のDiscord ID + 表示名
- 入退室時刻

用途:
- リピーター把握
- 常連の特定（機能差別はしない）

### 2.2 治安監査ログ（Pro: 30日）

保持される情報:
- モデレーションアクション（Kick/Mute/Ban）
- 実行日時
- 対象ユーザー
- 理由（入力された場合）

用途:
- 荒らし対策の振り返り
- 通報時のエビデンス

### 2.3 応援ログ（Pro: 30日、将来実装）

保持される情報:
- 応援日時
- 応援者（匿名 or 表示名）
- 金額
- メッセージ（あれば）

用途:
- 感謝の振り返り
- 配信改善の参考

## 3. 発言ログ

### 3.1 基本方針

**発言はデフォルトで保存しない。**

理由:
- プライバシー保護
- ストレージコスト
- 法的リスク軽減

### 3.2 ピン保存（PinnedMessage）

**配信者が明示的に保存したメッセージのみ永続化。**

| 項目 | 内容 |
|------|------|
| 保存者 | 配信者のみ |
| 対象 | 休憩チャットのメッセージ |
| 保持期間 | 無制限（Pro） |
| 上限 | 未決（→ open-questions.md） |

操作フロー:
1. 休憩チャット中にメッセージ表示
2. 配信者が「ピン」ボタンをクリック
3. PinnedMessage として保存
4. 後から「ピン一覧」で閲覧可能

## 4. データ削除

### 4.1 自動削除

| データ | Free | Pro |
|--------|------|-----|
| 参加履歴 | 即時削除 | 30日後に自動削除 |
| 治安監査ログ | 即時削除 | 30日後に自動削除 |
| セッションデータ | 7日後 | 30日後 |

### 4.2 手動削除

- 配信者は自分のルームのデータを手動削除可能
- ユーザーは自分のアカウントを削除可能（全データ削除）

### 4.3 アカウント削除時

| データ | 処理 |
|--------|------|
| User レコード | 削除 |
| Room（所有） | 削除 |
| SessionMember | userId を null に（匿名化） |
| ModerationAction | targetUserId を null に |
| Report | reporterId/targetUserId を null に |
